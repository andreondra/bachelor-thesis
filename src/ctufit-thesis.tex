\ctufitabstractCZE{Bakalářská práce se zabývá problematikou emulace v~kontextu výuky principů počítačových architektur a~jejich hardwaru. Na konkrétním příkladě zábavního systému Nintendo Entertainment System ukazuje celý proces vývoje softwarové simulace (emulátoru) od pochopení základních principů a~nutnou analýzu emulovaného systému přes návrh vhodného řešení na základě zjištěných informací až po samotnou implementaci v~jazyce C++ a~testování výsledku. Pro motivaci dalších zájemců o~problematiku je v~poslední kapitole obsažen výčet funkcionalit, které je možné doimplementovat.}
\ctufitabstractENG{Bachelor's thesis is focused on a~problematics of a~software emulation in a~context of teaching the principles of computer architectures and an associated hardware. There is a~whole emulator development process presented on an example of a~particular computer system, which is Nintendo Entertainment System. The process consists of understanding the basic principles behind software emulation, analysis of the emulated system, design of the solution based on discovered information and finally, the implementation of the emulator including testing. To motivate other students (or hobbyists) interested in the topic, there is a~list of possible extensions of the project in the last chapter of the thesis.}

%

%     \signal*{AD}           ->   AD#
%     \signal*[3::0]{C/BE}   ->   C/BE[3::0]#

		#3%
		\IfValueTF{#2}{[#2]}{}%
		\IfBooleanTF{#1}{\#}{}%
}}


% Draw pixels from a list specified in the first argument with colors defined with
% with prefix specified in the second argument.
% Inspired by an answer by Paul Gaborit at https://tex.stackexchange.com/questions/157080/can-tikz-create-pixel-art-images.
\NewDocumentCommand{\drawpixels}{mm}{
	\foreach \line [count=\y] in #1 {
		\foreach \pix [count=\x] in \line {
			\draw[fill=#2 \pix] (\x,-\y) rectangle +(1,1);
		}
	}
}
